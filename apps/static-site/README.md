# Mission Control — static replay snapshot

A self-contained, **read-only** build of the Mission Control dashboard that
**replays a day** of Panopticon activity from baked JSON instead of a live
server — no auth, no backend. Drop it on any static host (Vercel, Netlify, S3).

Everything in this directory except `README.md` and `.gitignore` is generated by
the export script and is **git-ignored** — regenerate it, don't edit it.

## Generate

With the Panopticon server running (so the export can read it):

```sh
node scripts/export-static-snapshot.mjs
# options:
#   --repo  <repo>   scope to a repository (default: fml-inc/panopticon)
#   --since <ISO>    earliest session start to include (default: today 00:00 local)
```

This writes `index.html`, `app.js`, `style.css`, and `data/` here.

### Scope = the privacy boundary
The export builds an **allowlist** of sessions: those in `--repo` started on/after
`--since`, plus their subagents (transitively). Only allowlisted sessions,
their timelines, and in-room bus messages are written — nothing outside the
allowlist can leak. Content is **not** redacted, so only widen the scope to data
you're comfortable making public.

## Preview locally

```sh
npx serve apps/static-site      # or: python3 -m http.server -d apps/static-site 8099
```

The dashboard loads in **replay mode**: it autoplays the day on a virtual clock
(roster filling in, bus challenges firing, delivery landing). Controls: play/pause,
restart, scrubber, speed (1–8×), and jump-to-end. The operator console and live
SSE are disabled (it's a frozen, read-only snapshot).

## Deploy

```sh
vercel deploy apps/static-site --prod
```

Any static host works — it's just `index.html` + `app.js` + `style.css` + `data/`.
