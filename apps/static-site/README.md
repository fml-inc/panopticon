# Mission Control — shareable snapshot

A self-contained, **read-only** build of the dashboard that **replays a day** of
Panopticon activity on a virtual clock (play/pause, scrubber, speed, jump-to-end).
Everything here except `README.md` / `.gitignore` is generated by the export
scripts and is **git-ignored** — regenerate, don't edit.

Both flavors scope strictly to **today's panopticon effort** (repo + `--since` +
subagents) as an allowlist — nothing outside it is written. Content is not
redacted, so only widen scope to data you're fine making public.

## Two ways to ship it

### A) DB-backed (recommended) — fully queryable, runs over a copy of the DB
A small scoped SQLite copy + a Vercel serverless `/api/tool` that reuses the real
query code. Every session/timeline + arbitrary query works on the public link.

```sh
pnpm build                              # builds dist/ (incl. dist/service)
node scripts/export-scoped-db.mjs       # → apps/static-site/db/panopticon.db + assets
node scripts/serve-snapshot.mjs         # preview locally at :8787
vercel deploy --prod                    # deploy (see vercel.json / api/tool.js)
```

Caveats: cost-as-of-T isn't wired yet (OTel-derived), so the spend tile stays
blank while scrubbed. The Vercel function relies on `better-sqlite3` linux
prebuilds at runtime — if the platform rejects it, switch to `node:sqlite`/libSQL.

### B) Static JSON — zero backend
Bakes the in-scope rows to JSON; no functions. Roster/feed/timeline replay over
baked data; no arbitrary query, no cost-as-of-T.

```sh
node scripts/export-static-snapshot.mjs   # → index.html + app.js + style.css + data/
npx serve apps/static-site                 # preview
vercel deploy apps/static-site --prod      # deploy (pure static)
```

## Replay controls
Autoplays the day; play/pause, restart, scrubber, speed (10× / 100× / 1000×),
and jump-to-end. The operator console and live SSE are disabled (read-only).
