### Summary
Two logically distinct changes to `src/frenemy/driver.ts`:

2. **A feature graft** — `gitDiff` now falls back from working-tree diff (`diff HEAD`) to this branch's committed work vs its base (`diff <base>...HEAD`) when the worktree is clean, so committed/sibling-worktree work is still reviewed instead of yielding an empty diff. The original bug was the *process* exiting mid-settle. A vitest run keeps the process alive regardless, so the test can't observe that failure mode. The test is genuinely useful — it proves the loop reaches the review pass after the settle wait and stops cleanly — but the comment "(A previously unref'd settle timer let the process exit mid-settle.)" claims a guard the test cannot provide. Consider softening the comment, or, if you want a true guard, assert on the handle's structure rather than implying process-exit coverage. Not a blocker.

**2. `gitDiff` branch-scope quality depends on `origin` being fetched (minor).**

If origin is stale, `merge-base(origin/main, HEAD)` can sit far back and the three-dot diff balloons (then gets truncated at 8000 chars, possibly hiding the actual recent change behind unrelated older commits).
